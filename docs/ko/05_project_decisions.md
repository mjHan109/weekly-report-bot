# 프로젝트 확정 결정사항

> **문서 동기화 규칙:** `docs/en/05_project_decisions.md`와 동기화한다.

## 1. 보고 주간 정의
- **확정:** **이전 주 목요일 13:00:01 ~ 이번 주 목요일 13:00:00 (KST)**
- **정식 마감:** 매주 **목요일 13:00 (KST)** — on-time 제출 기준 시각
- 경계: KST(Asia/Seoul)

### 1.1 보고 대상 (확정)
- **팀장이 채널별로 보고 대상을 지정** (`보고 대상 지정`)
- 저장: `ChannelReportTarget` / `ChannelConfig.report_target_user_ids`
- 알림·제출·취합·메일 게이트는 **지정 대상만** 기준

### 1.2 13:00 이후 제출 (확정 — 대리 제출 없음)
- **팀장 대리 제출은 사용하지 않는다.**
- **13:00 시점 미제출 지정 대상**은 마감 이후에도 **본인**이 `이번 주 보고 작성`으로 제출할 수 있다. (late self-submit)
- 이미 on-time 제출한 대상은 재제출 불필요 (수정은 기존 수정 플로우)
- late 제출: `submitted_after_deadline=true` 이력 저장

### 1.3 목 10:00 알림
- **채널** 메시지 — 지정 대상 중 **미제출자** 대상, 13:00 마감 안내

### 1.4 목 13:00 마감 처리
#### A. 지정 대상 전원 on-time 제출 (13:00 이전)
- **자동 취합** (팀장 명령 불필요)
- 팀장에게 **Adaptive Card**: 취합 미리보기 + **`메일 작성`** 등 후속 액션

#### B. 13:00 시점 미제출자 있음
- **채널** 메시지: 미제출자에게 작성 요청 (본인 `이번 주 보고 작성` 가능)
- **팀장에게 Adaptive Card** (채널 또는 1:1 — Phase 1, 기본 채널 권장):
  - 미제출자 명단
  - **「아직 N명 미제출 · 제출 대기 중 · 메일 발송 불가」**
  - (선택) 미제출자에게 제출 독려 안내
- **자동 취합 없음**, 메일 차단
- 미제출 대상의 **본인 제출을 기다림**

#### C. late 제출 후 전원 제출 완료
- 지정 대상 **전원 제출**이 완료되면 (late 포함) **팀장에게 Adaptive Card** 갱신/재발송:
  - **「전원 제출 완료 · 취합 가능」**
  - **`팀 주간 보고 취합`** 버튼 (수동 취합)
- 팀장이 취합 실행 → 취합 **Adaptive Card** 미리보기 → **`메일 작성`** / `발송` 플로우
- **13:00에 미제출이 있었거나 late 제출이 1건이라도 있으면** 자동 취합 없이 **수동 취합**만 허용

### 1.5 메일 발송 전제
- 지정 대상 **전원 제출** + **취합 완료** + **팀장 발송 승인**
- 미제출자 1명이라도 있으면 메일 불가

## 2. 입력 UI
- **Task Module** — 지정 대상 본인 작성 (on-time / late 동일)
- **Adaptive Card** — 개인·취합 미리보기, **팀장 상태/후속 액션 카드**

## 3. 팀 취합
| 조건 | 취합 |
|---|---|
| 13:00 전 지정 대상 전원 on-time 제출 | **자동** |
| 13:00 미제출 있음 또는 late 제출 있음 | **수동** (`팀 주간 보고 취합`, 팀장만) |

## 4. 팀장 등록
- `팀장 등록` — **최초 관리자** + **팀장 본인**만

## 5. 메일 To/CC · Graph · 채널
- (기존 확정) Delegated OAuth, 채널 격리, To/CC 팀장 지정
